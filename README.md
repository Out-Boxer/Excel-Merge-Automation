# 주요 기능 및 목적
- 엑셀 표 병합(최대한 원본 서식 유지)
- 여러 개의 명단이나 표 리스트 등을 하나의 엑셀로 병합 시 일일이 복사 붙여넣기 하는 과정 귀찮음 해소
- 붙여넣기 시 무한 로딩 및 엑셀 멈춤 현상 및 멈춤 현상으로 인한 엑셀 비정상 종료 이슈 방지 목적
- 제작 당시에는 AI 에이전트나 AI 비서가 사용되기 전 시절이나 현재 AI 관련 보안 이슈가 지속적으로 제기되는 상황에 표 병합을 로컬에서 진행하여 혹시 모를 최대한 보안 사고를 최소화

# 제작 동기
- 공공기관에서 사무직 근무 당시 복사 붙여넣기 반복 작업 최소화 목적 (민원인의 방문 및 민원 전화가 잦은 부서라 시간 효율 중요!)
- PC 사양이 제각각이라 엑셀 대량 병합 시 저사양 PC에서의 문제는 물론이거니와 양이 많은 경우 적정 사양 PC에서도 문제 발생 >> 엑셀 복사 붙여넣기 로딩 문제는 유명
- 붙여넣기 시 무한 로딩 및 엑셀 멈춤 현상 및 엑셀 비정상 종료 이슈 해소(pc 사양에 따라 차이가 있지만 로딩 10분 이상 걸린 경우 O)
- 민원 전화가 왔는데 엑셀 로딩 이슈로 PC 사용 어려움을 겪는 경우 다수

# 사용한 기술
- 당시 최신 기술이었던 웹 제미나이 2.5 pro 바이브 코딩(제작 당시) 
- 웹 GPT 최신 버전 or GPT 코덱스 최신 버전 (앞으로의 리팩토링 시 사용할 AI)
- 파이썬 3.X
- 파이썬 openpyxl 라이브러리
- 파이썬 tkinter GUI
- 큐 자료구조

# 굳이 웹 AI 사용을 고수한 이유
- 제작 당시에도 안티그래비티나 커서 등이 존재했으나 다루는 파일들이 개인 정보를 포함한 파일들이라 공공기관 특성을 고려한 혹시 모를 보안 사고를 방지하기 위해 웹에서 코드 복사 붙여넣기 방식 고수
